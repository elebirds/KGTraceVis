# brainstorm: unify TEP KG construction and RCA narrative

## Goal

Understand how the TEP Root-KGD runtime files used by KGTraceVis are generated in `~/code/TEP_KG`, then propose a coherent paper and system narrative that unifies source-constrained KG construction with RCA analysis.

## What I already know

* KGTraceVis currently loads standard TEP KG CSV files for canonical KG reasoning.
* TEP Root-KGD inference also loads additional runtime assets under `data/kg/tep_root_kgd`.
* Those extra assets include graph views, variable mappings, relation parameters, edge weights, anchor discriminators, and anchor memory profiles.
* The current KG compiler exposes a `runtime_views_manifest` output, but it is currently a placeholder.

## Assumptions (temporary)

* `~/code/TEP_KG` contains the original generation scripts or intermediate artifacts for the Root-KGD assets.
* Some runtime assets are deterministic projections from source/KG data, while others are calibrated from TEP simulation windows.
* The final narrative should avoid claiming that all RCA assets are plain KG facts.

## Open Questions

* Which Root-KGD files can be generated from the source KG compiler, and which require a separate calibration/evaluation pipeline?

## Requirements (evolving)

* Trace each runtime asset to its generation source in `~/code/TEP_KG`.
* Classify runtime assets into canonical KG, runtime projection, detector evidence, and calibration/model assets.

## Technical Notes

* KGTraceVis TEP Root-KGD loader: `src/kgtracevis/workflows/tep_root_kgd/__init__.py`.
* KGTraceVis TEP producer emits Root-KGD contributions and dynamic features: `src/kgtracevis/producers/tep_records.py`.
* `TEP_KG/scripts/build_kg.py` calls `tep_kg.graph_build.build_kg`, which writes Full KG, semantic lift, TEP variable mapping, and RCA graph projection outputs.
* `TEP_KG/data/processed/rca/nodes.jsonl` and `edges.jsonl` are generated by `build_rca_graph` during `build_kg`.
* `TEP_KG/data/processed/kg/tep_variable_mapping.jsonl` is generated by `build_tep_variable_mapping` during `build_kg`.
* `TEP_KG/data/processed/rca/anchor_discriminators.json` is generated by `scripts/build_anchor_discriminators.py`; it combines seed-curated rows and LLM-assisted diagnostic variable selection over RCA anchors.
* `TEP_KG/data/processed/models/rca_edge_weights.jsonl` and `relation_family_params.json` are generated by `tep_kg.edge_weight_training.build_edge_weight_training`.
* `TEP_KG/outputs/rca/anchor_memory_profiles.json` is generated from RBC scenarios plus dynamic features by `build_anchor_memory_profiles`.
* SHA-256 checks confirm KGTraceVis `data/kg/tep_root_kgd/*` files are byte-identical to their TEP_KG source artifacts.
